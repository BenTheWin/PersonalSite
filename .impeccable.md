## Design Context

### Users
Recruiters and potential employers reviewing Ben's portfolio. Also peers/developers who Ben shares the link with. Context: desktop-primary, viewed during job search browsing. The job to be done: quickly understand who Ben is, what he can build, and how to contact him.

### Brand Personality
Bold · Genuine · Creative

Ben is a recent software engineering grad who loves purple, plays drums, reads manga, and games. The site should feel like a person with real taste — not a corporate template, not "developer gray." It should be memorable and a little unexpected.

### Aesthetic Direction
Retro-pop bold — inspired by physical "Hello My Name Is" nametag badges and stacked manila file folders. Heavy black borders everywhere (3px solid #111). Chunky typography. Three distinct section "folders" that visually stack and overlap like real folders on a desk.

**Palette (exact values — do not deviate):**
- Purple: `#9333EA` — primary accent, About folder bg, skill tiles, CTA buttons
- Cream: `#F0EAD6` — page bg, Skills folder bg, nametag inner box
- Green: `#16A34A` — Projects folder bg, Live link buttons
- Black: `#111111` — all borders, body text, nav, footer bg

**Anti-references:** No dark mode. No glassmorphism. No gradient text. No generic developer gray. No Inter or similar "safe" fonts.

**Theme:** Light (cream base). Bright, vibrant, and energetic.

### Design Principles
1. **Every border is intentional** — 3px solid black is the visual signature of this site. Consistent everywhere.
2. **Folders, not sections** — About/Skills/Projects are physical folder cards with protruding rounded tabs, not generic section dividers.
3. **Typography punches hard** — The name badge uses massive uppercase type. No decorative gradients. Weight and size create all hierarchy.
4. **Color has territory** — Each folder owns its color. Purple = About, Cream = Skills, Green = Projects. Never bleed across.
5. **Restraint in motion** — Subtle hover states only. No scroll animations that feel gimmicky.
